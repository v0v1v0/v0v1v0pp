guardduty_list_filters R Documentation

Returns a paginated list of the current filters

Description

Returns a paginated list of the current filters.

See https://www.paws-r-sdk.com/docs/guardduty_list_filters/ for full documentation.

Usage

guardduty_list_filters(DetectorId, MaxResults = NULL, NextToken = NULL)

Arguments

DetectorId

[required] The unique ID of the detector that the filter is associated with.
MaxResults

You can use this parameter to indicate the maximum number of items that you want in the response. The default value is 50. The maximum value is 50.
NextToken

You can use this parameter when paginating results. Set the value of this parameter to null on your first call to the list action. For subsequent calls to the action, fill nextToken in the request with the value of NextToken from the previous response to continue listing data.
